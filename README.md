# Website Project Template

Fork this repo at the start of every new project. Fill in each doc in order — discovery first, everything else follows from it.

## Docs

| File | When to fill |
|------|-------------|
| [docs/discovery.md](docs/discovery.md) | First client call |
| [docs/content-guide.md](docs/content-guide.md) | During discovery — pick the right sections for the client type |
| [docs/style-guide.md](docs/style-guide.md) | After discovery, before design |
| [docs/tech-stack.md](docs/tech-stack.md) | After discovery, before dev |
| [docs/pricing.md](docs/pricing.md) | Before sending the quote |
| [docs/handoff.md](docs/handoff.md) | Pre-launch and delivery |

## How to use

1. Fork this repo, rename it to the client/project name
2. Work through `discovery.md` during the first call
3. Fill in the remaining docs as the project progresses
4. Use `handoff.md` as your go-live checklist
