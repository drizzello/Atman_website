# Handoff & Launch Checklist

Work through this before going live. Check off each item.

---

## Pre-Launch

### Content
- [ ] All pages have final copy (no placeholder text)
- [ ] All images are optimized (WebP, correct sizes)
- [ ] Favicon added
- [ ] 404 page exists

### SEO & Meta
- [ ] Page titles set for all pages
- [ ] Meta descriptions set for all pages
- [ ] Open Graph image set (for link previews)
- [ ] `sitemap.xml` generated and accessible
- [ ] `robots.txt` configured

### Performance
- [ ] Lighthouse score checked (target: 90+ on mobile)
- [ ] Images lazy-loaded
- [ ] No render-blocking resources

### Functionality
- [ ] All links work (no broken hrefs)
- [ ] Contact form tested (sends and receives correctly)
- [ ] Any integrations tested (booking, maps, payments)
- [ ] Site tested on mobile (iOS + Android)
- [ ] Site tested on desktop (Chrome, Safari, Firefox)

### Analytics
- [ ] Google Analytics / Plausible set up
- [ ] Tracking verified (at least one pageview recorded)

---

## Domain & Hosting

- [ ] Domain purchased and confirmed
- [ ] DNS records pointed to hosting (A record / CNAME)
- [ ] SSL certificate active (HTTPS working)
- [ ] `www` redirect configured (www → non-www or vice versa)
- [ ] Hosting deployment connected to repo (if using Vercel/Netlify)

---

## Delivery

- [ ] Final invoice sent
- [ ] Login credentials / access handed to client (if applicable)
- [ ] Client briefed on how to request future changes
- [ ] Repository access transferred or archived

---

## Post-Launch Notes

Any issues found after launch, client feedback, or pending items.
